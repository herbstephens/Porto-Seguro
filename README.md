# Porto Seguro Lisboa 🌿
### Emergency shelter & support app for domestic violence victims in Lisbon, Portugal
**PWA · Works offline · Bilingual PT/EN · Disguised as a weather app on home screen**

---

## Deploy in 15 minutes — step by step

### 1. Install dependencies locally
```bash
npm install
npm run dev     # test at http://localhost:5173
```

### 2. Create a GitHub repository
1. Go to https://github.com/new
2. Name it `porto-seguro` (keep it **private**)
3. Do NOT initialise with README (you already have one)
4. Copy the remote URL shown

### 3. Push the code
```bash
git init
git add .
git commit -m "Initial Porto Seguro PWA"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/porto-seguro.git
git push -u origin main
```

### 4. Deploy to Vercel (free)
1. Go to https://vercel.com → sign in with GitHub
2. Click **Add New → Project**
3. Import the `porto-seguro` repository
4. Framework preset: **Vite** (auto-detected)
5. Build command: `npm run build`
6. Output directory: `dist`
7. Click **Deploy** → done in ~90 seconds

Your app is live at: `https://porto-seguro.vercel.app`

### 5. Add a custom .pt domain (recommended)
1. Buy `portoseguro.pt` or `ajuda-ja.pt` from a Portuguese registrar
   e.g. https://www.fccn.pt/dominios-pt/ (official PT registry)
   or https://www.namecheap.com
2. In Vercel → Settings → Domains → Add domain
3. Add the DNS records shown (CNAME or A record)
4. HTTPS is automatic — Vercel handles Let's Encrypt

## Making updates
Any `git push` to `main` triggers an automatic redeploy on Vercel.

```bash
# Edit a file, then:
git add .
git commit -m "Update shelter data"
git push
# Live in ~60 seconds
```

## PWA: How users install it
### Android / Chrome
- Banner appears automatically after ~30 seconds of use
- Or: browser menu → "Add to Home Screen"
- App installs silently, appears as "Tempo" on home screen

### iPhone / Safari
- User taps the Share button (⎙)
- Taps "Add to Home Screen"
- App shows as "Tempo" icon

### The disguise
The app appears as **"Tempo"** (weather) on the phone home screen.
The icon looks like a sun/cloud. The internal title shows the real name.
This protects users from an abuser recognising the app.

## Offline support
The service worker (auto-generated by vite-plugin-pwa) caches:
- All app HTML/JS/CSS
- All data (hotlines, shelters) is embedded — no external API calls
- The app works fully with **zero internet connection**

---

## Security headers (already in vercel.json)
- `X-Frame-Options: DENY` — prevents embedding in iframes
- `Referrer-Policy: no-referrer` — doesn't leak the URL to third parties
- `Content-Security-Policy` — blocks external script injection
- Service worker served with `no-cache` to ensure instant updates

---

## Updating shelter / hotline data
All data lives in `src/App.jsx` at the top of the file in the
`HOTLINES`, `SHELTERS`, `HOSPITALS`, and `POLICE_STATIONS` arrays.
Edit those arrays and push — no database needed.
